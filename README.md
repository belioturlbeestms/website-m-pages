# website-m-pages

这是一个用于归档和发布多个独立 HTML 页面的仓库。  
页面内容可涵盖演示、工具、存档、静态展示等场景，不针对任何特定域名或网站。

## 目录说明

```
website-m-pages/
├── pages/           # 存放所有归档的 HTML 页面
├── assets/          # 页面引用的公共资源（图片、样式、脚本等）
├── index.html       # 可选：仓库首页或页面列表
└── README.md        # 本文件
```

- `pages/` 目录下每个 HTML 文件代表一个独立页面，命名应简洁、无歧义。
- `assets/` 用于存放页面共享的静态资源，避免重复。
- 仓库根目录下的 `index.html` 可作为一个导航或说明页，非必需。

## 页面归档说明

- 每个页面应保持独立，尽量不依赖外部资源，以确保离线或本地查看时可用。
- 页面之间不建议相互引用，避免形成依赖链。
- 文件命名建议使用英文小写字母、数字、连字符（`-`）或下划线（`_`），避免特殊字符。
- 若页面包含外部链接或引用，建议在页面内注明来源或用途。

## 维护说明

- 新增页面时，将 HTML 文件放入 `pages/` 目录，并检查资源引用路径是否正确。
- 修改或删除页面时，请同步更新相关索引或说明（如有）。
- 欢迎提交 Pull Request 或 Issue 讨论页面归档内容。
- 本仓库不承诺长期在线托管，建议自行备份重要页面。
